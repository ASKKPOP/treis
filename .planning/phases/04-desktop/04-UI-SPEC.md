# Phase 4 — UI Design Contract

> Visual and interaction contract for the Treis Electron desktop app. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This is a developer-tools macOS app. The visual language follows that constraint: dark surface, monospaced data, minimal chrome. No marketing gradients. No decorative illustration.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — Tailwind 4 utility classes directly) |
| Preset | not applicable |
| Component library | @headlessui/react 2.2.10 (violation modal only) |
| Icon library | lucide-react (MIT, tree-shakeable, consistent stroke weight) |
| Font | Inter (sans, system-ui fallback) for UI text; JetBrains Mono for code/tool output/trace data |

Source: RESEARCH.md Standard Stack — Tailwind 4.2.2 and @headlessui/react 2.2.10 are declared; icon library and fonts are researcher defaults for developer tools.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Between label and value, list item gaps |
| md | 16px | Card internal padding, form field spacing |
| lg | 24px | Screen section padding, card-to-card gaps |
| xl | 32px | Screen lateral padding (left/right gutters) |
| 2xl | 48px | Top/bottom page breathing room |
| 3xl | 64px | Vertical centering margin on focused screens (IntentInput) |

Exceptions:
- Touch / click target minimum: 44px height for all interactive controls (button, card, input)
- Plan option cards (DESK-06): fixed height 160px to maintain 3-card visual parity
- Step counter row (DESK-08): 40px row height for compact execution stream

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px | 400 | 1.5 | Inter | General prose, descriptions, dialogue text |
| Label | 12px | 500 | 1.4 | Inter | Field labels, metadata, step counters, badges |
| Heading | 20px | 600 | 1.2 | Inter | Screen titles, card titles, modal headings |
| Mono | 13px | 400 | 1.6 | JetBrains Mono | Tool output, trace tokens, scope entries, file paths |

Rules:
- Exactly 2 weights in use: regular (400) and semibold (600). The 500 label weight is a nudge-up of 400 — within the "two weight" spirit for data-density contexts.
- No italic. No underline except hyperlinks (none in Phase 4).
- Mono role is reserved for machine-generated content: streaming tokens, tool call arguments, file paths, scope boundary entries. Never use mono for UI copy.

## Color

Palette is dark-mode only. Treis is a macOS developer tool — dark is the expected default. No light mode in Phase 4.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F1117` | App window background, screen backgrounds |
| Secondary (30%) | `#1A1D27` | Cards, sidebar (if any), modal backdrop, input fields, plan option cards |
| Accent (10%) | `#7C6AF7` | — see reserved list below — |
| Surface-border | `#2A2D3E` | Card borders, dividers, input borders, step separators |
| Text-primary | `#F0F0F5` | All primary readable text |
| Text-secondary | `#8B8FA8` | Metadata, secondary labels, disabled state |
| Destructive | `#E5534B` | Stop action, violation interrupt badge, error states |
| Success | `#3FB950` | Pass criterion checkmarks, completed step indicators |
| Warning | `#F0A04B` | WARN budget exceeded, escalation prompt |

Accent (`#7C6AF7`) reserved for:
- The primary CTA button on each screen (Begin Execution, Send, Select Plan)
- The active/focused state ring on the intent input field
- The selected plan option card left border
- The current executing step indicator dot

Accent is NOT used for: headings, body text, icons in resting state, dividers, backgrounds.

Destructive (`#E5534B`) reserved for:
- The "Stop" button in the violation modal
- Contract violation interrupt badge/banner
- Error state inline messages
- FATAL circuit-breaker alert

## Screen Inventory and States

### Screen 1: IntentInput (DESK-04)

**Layout:** Centered single-column, vertically centered at 33% of viewport height. Max-width 560px.

**States:**
- **idle** — text input unfocused, placeholder visible, "Start" button disabled
- **focused** — accent ring on input, "Start" button enabled when input non-empty
- **submitting** — input disabled, button shows spinner (16px), no label change

**Required elements:** Single `<textarea>` (auto-resize, max 4 rows), primary CTA button.

### Screen 2: Dialogue (DESK-05)

**Layout:** Single-column, max-width 640px, centered. Scrollable message list with input pinned to bottom.

**States:**
- **ai-typing** — streaming tokens appear in the AI message bubble; input disabled with placeholder "AI is thinking..."
- **awaiting-input** — user input field enabled, send button active
- **complete** — final AI message rendered, "Review Plans" button appears below last message

**Required elements:** Message bubbles (AI = secondary surface, User = accent-tinted surface), streaming token display (mono font), fixed-bottom input + send button.

### Screen 3: PlanOptions (DESK-06)

**Layout:** Three equal-width cards in a row. Each card: fixed 160px height, 16px internal padding, left accent border (4px) on hover/selected state.

**States:**
- **unselected** — card with surface-border, no accent
- **hovered** — 2px accent left border, slight secondary surface brightening
- **selected** — 4px accent left border, slight secondary surface tint

**Required elements:** Card with badge (A/B/C), plan name (heading), tradeoff description (body), keyboard shortcut hint (label).

### Screen 4: SealedContract (DESK-07)

**Layout:** Single-column, max-width 640px. Sections: scope entries list, boundaries list, success criteria list. "Begin Execution" button pinned to bottom of content (not viewport).

**States:**
- **reviewing** — read-only display, "Begin Execution" button active
- **beginning** — button disabled + spinner while worker thread initializes

**Required elements:** Section labels (heading), mono-font scope entry rows, success criteria list (unchecked checkboxes, decorative only — not interactive here), "Begin Execution" primary CTA.

### Screen 5: ExecutionStream (DESK-08)

**Layout:** Two-panel split. Left panel (60%): step-by-step event stream, scrolls automatically to latest. Right panel (40%): current step detail / tool call args + result.

**States:**
- **running** — new events appending, step counter incrementing, latest step highlighted with accent dot
- **violated** — execution pauses, ViolationModal overlays (see below)
- **escalating** — escalation prompt appears inline in stream as a distinct row
- **complete** — stream stops, "View Results" button appears

**Required elements:** Step row (step number label, state icon, step description), tool call row (tool name mono, args mono collapsed), tool result row (collapsed by default, expand on click), evaluator verdict badge (PASS/FAIL/RETRY).

### Screen 6: Result (DESK-09)

**Layout:** Single-column, max-width 640px. Deliverable summary at top, success criteria checklist below.

**States:**
- **success** — all criteria checked in success green, heading "Done"
- **partial** — mixed pass/fail, heading "Completed with issues"
- **failed** — all criteria failed, heading "Execution failed"

**Required elements:** Success criteria list items with pass/fail icon (success green check or destructive X), deliverable summary text (body), "Start New Task" secondary CTA.

### ViolationModal (DESK-05, DESK-08)

**Layout:** Centered modal overlay with backdrop. Max-width 480px.

**States:**
- **open** — rendered over whichever screen is active; execution paused
- **resolving** — user clicked an option; buttons disabled

**Required elements:** Violation description (body), three action buttons — "Stop" (destructive), "Amend" (secondary), "Continue Differently" (secondary). Implemented with @headlessui/react Dialog. Focus trap active. Escape key does NOT close (user must make a choice).

## Interaction Contracts

### Keyboard Navigation
- Tab order: standard DOM order, no custom tabIndex jumps
- Enter submits focused form (intent input, dialogue input)
- A / B / C keys select plan option on PlanOptions screen
- Escape does NOT close ViolationModal (user must choose)
- No global keyboard shortcuts in Phase 4

### Focus Management
- On screen transition: focus moves to the first interactive element of the new screen
- On ViolationModal open: focus moves to first button in modal (not Stop — Amend is first)
- On ViolationModal close: focus returns to the element that triggered the modal

### Loading / Async States
- Buttons in submitting state: replace label with 16px spinner (Tailwind `animate-spin`), keep button width fixed to avoid layout shift
- Streaming tokens: append-only, no re-render of previous content
- ExecutionStream: auto-scroll to latest event unless user has scrolled up (detect scroll position, pause auto-scroll)

### Sizing
- Minimum window size: 900px × 600px (enforced via `minWidth`/`minHeight` in BrowserWindow)
- Maximum responsive width for content columns: 640px (capped, centered)
- App is not resizable below minimum — no responsive breakpoints required for Phase 4
